Continued work and tried to make the ai agents specify more on the entities encountered.

Expanded entity set from 5 classes to 40 classes (11 Animal, 9 Resource, 10 Danger, 5 CraftOpp, 5 Event). 

- The 5-class reconstruction was too easy, the agent achieved 100% recon but only had 5 unique messages (one per class). With only 5 possible messages, the receiver wasn't getting enough information to make good action decisions.

Then to accommodate this change I also changed the recon head:

- Changed the reconstruction head from `Linear(128→64→5)` (5 entity classes) to `Linear(128→64→40)` (40 individual entities)
- `recon_weight` set to 1.5 (from 2.0)
- `reward_scale` set to 0.5 (from 0.2)
- `temperature` = 2.0, `temp_decay` = 0.975

Higher `reward_scale` (0.5 vs 0.2) was meant to give more weight to the reward component of the loss.

Lower `recon_weight` (1.5 vs 2.0) was a slight reduction to balance the higher reward scale.

This was a success, run 5 proved to have 100% recon acc by epoch 22 (vs epoch 3-5), generating 40 unique messages, so every entity has its own message. Despite this, the action policy was still not good, with a range of 32% to 40% survival rate.

Potential cause for this is that when the action logits go through the SoftMax without temp scaling, one action dominates and the SoftMax output becomes a near one hot vector. Once the gradient through the reward term is effectively zeroed out for non-dominant actions, exploration stops.

Tried to fix this action policy problem via three directions

1. Added an action entropy bonus to the loss which penalises low entropy (peaked) distributions, which coeff = 0.1
2. Added an action temperature, going from SoftMax(z) to SoftMax(z/t), with t = 2.0 for the test.
3. Normalized the rewards, so for each sample the 11 action reward vector is normalized to zero mean and unit variance, making the gradient equally strong for all samples regardless of absolute reward magnitude.

Without temperature scaling, the SoftMax concentrates probability mass on one action very quickly. Without entropy bonus, there's no incentive to maintain exploration. Without reward normalisation, samples where all actions give similar rewards produce near-zero gradients.

Run 6 proved this right, with an increase in the survival rate of a mean of 41% and a peak of 49%. Although i need to run it longer as the recon acc got to 96.7% only in 50 epochs.

 The action policy is stuck in a local optimum. The agent has learned to communicate (especially in Run 5 with perfect reconstruction), but cannot translate entity knowledge into context-dependent actions. Gather/eat always yields safe, positive reward, risky context-dependent actions (hunt, flee) require coordinating entity-type understanding with the correct action, and the gradient signal for this is too weak relative to the stable gather/eat baseline. 

Run 7 and 8 where experiments on hyperparameter tunning, 7 had no temperature decay and 8 was a failure.

Run 7:

1. **Higher entropy** (`action_entropy_coeff=0.5`): 5× stronger exploration pressure
2. **Lower action temperature** (`action_temperature=1.0`): Standard softmax (no temperature scaling), so the agent's logit differences directly translate to probability differences
3. **No temperature decay** (`temperature_decay=0.0`): Keep GS communication temperature at 1.0 throughout. No annealing.
4. **Lower initial GS temperature** (`temperature=1.0`): Start with a sharper communication channel from the beginning (vs 2.0 which starts soft and anneals)
5. **Longer training** (70 epochs vs 50)

The results gave a survival mean of 39.4% with a peak of 49% but a much higher variance. There was also a problem with the recon acc as it only got to 52.2% with 26 messages, but having the same action distribution as before. This proved that abusing the entropy bonus damages the recon loss and competes with it, making the model not learn good communication and maintain exploration simultaneously.

**Structure & compositionality**

Relevant metrics

- posdis (positional disentanglement)
- bosdis
- TopSim

These measure whether message structure corresponds to semantic structure.

**Positional disentanglement (posdis):**

Mean:

- Run 11: 0.013
- Run 12: 0.039

Final:

- Run 11: 0.0178
- Run 12: 0.0136

Values close to 0 mean almost no compositional structure.

So in both runs:

- communication is largely holistic
- messages behave like atomic symbols

Run 12 slightly higher mean posdis suggests minor structural emergence, but still extremely low.

**Bag-of-Symbols Disentanglement (bosdis):**

Checks:

Does a **symbol consistently encode one specific attribute?**

Mean:

- Run 11: low (not strongly structured)
- Run 12: higher than Run 11 (stronger signal)

Final:

- Run 11: low
- Run 12: low–moderate

Interpretation:

- Values are still relatively low overall
- However, higher bosdis (especially in Run 12) suggests:
    - some symbols are reused systematically
    - weak **category-level encoding** (e.g. same token for similar entities)

So:

- not compositional
- but not fully random either

This indicates **proto-structure at the symbol level**, likely acting as prefixes or category markers.

**Topographic Similarity (TopSim):**

Ran on a sub sample of 2000 per epoch

Mean:

- Run 11: ~0.30–0.33
- Run 12: ~0.32–0.35

Final:

- Run 11: ~0.30
- Run 12: ~0.32

TopSim measures correlation between:

- distance in meaning space
- distance in message space

Interpretation:

- Values around **0.3** indicate **moderate alignment**
- Far from 1 → not compositional
- Above 0 → not random

So in both runs:

- semantically similar entities tend to have somewhat similar messages
- but this structure is weak and inconsistent

This matches a **clustered or category-based signalling system**, rather than a fully compositional language.

**Overall interpretation:**

- posdis ≈ 0 → no positional compositionality
- bosdis > 0 → weak symbol-level regularities
- TopSim ~0.3 → moderate semantic alignment
